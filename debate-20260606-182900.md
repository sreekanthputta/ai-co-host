## 3. Riskiest technical part and how to de-risk it

The riskiest part is **not Moss**. Moss is probably the least scary piece if the API works.

The riskiest part is the real-time pipeline:

```txt
Live audio → STT/partial transcript → retrieval → Minimax judgment → whisper generation → audio route to founder only
```

Your “380ms” claim is dangerous because Moss might be <10ms, but the total path is dominated by:

- STT partial latency
- chunking/VAD timing
- Minimax response latency
- TTS/voice playback latency
- browser/audio routing weirdness
- stage WiFi

If you put “380ms” on a big counter and it comes in at 2.4s, you look fake.

### De-risk plan

Build the demo around **“before the human responds”**, not “400ms mid-sentence.”

Use this target:

> Echo responds within 1 second of the risky phrase appearing in the transcript.

That is still impressive and actually demoable.

Implementation discipline:

1. **Preload a tiny memory corpus**
   - 8 prior call snippets
   - 1 pitch deck fact sheet
   - 5 objection examples
   - 3 competitor/customer snippets if you really need flavor

2. **Precompute Moss indexes**
   - Do not ingest live PDFs on stage.
   - Do not scrape competitors live.
   - Do not embed during the demo unless sponsor requires it.

3. **Use partial transcript chunks**
   - Every 500-800ms, send the latest sentence fragment to Moss.
   - Retrieve top 3-5 relevant snippets.

4. **Minimax does only the judgment**
   - Prompt it with:
     - current utterance
     - retrieved snippets
     - call context
   - Force JSON:
     ```json
     {
       "interrupt": true,
       "risk": "contradiction",
       "confidence": 0.91,
       "whisper": "Clarify: $250 blended, $400 paid.",
       "evidence": ["prior_call_3", "deck_cac"]
     }
     ```

5. **Only interrupt on high confidence**
   - False positives kill this product.
   - In the demo, fewer interruptions feels smarter.

6. **Have a visual fallback**
   - If whisper audio fails, the founder sees a private LiveKit side panel:
     > “Clarify: $250 blended, $400 paid.”
   - The public screen still shows Echo catching it.
   - Do not make the whole demo depend on an earbud working on stage.

7. **Optional hotkey fallback**
   - Have one teammate trigger the “risky phrase detected” flow if STT flakes.
   - Judges do not care if your ASR was 300ms slower. They care that Moss/Minimax/LiveKit are visibly integrated.

# Where it’s weak right now

## 1. “400ms later” is probably bullshit

Moss may be <10ms. The full system will not be.

You can maybe get:

- partial STT in 300-800ms
- Moss in <10ms
- Minimax in 300-1200ms depending on model/API
- TTS in 200-800ms

That puts you closer to 1-2 seconds unless heavily optimized.

Do not promise 380ms unless you have measured it by hour 8.

Better demo metric:

> “Retrieval: 8ms. End-to-end nudge: 820ms.”

That is honest and still sponsor-friendly.

## 2. “Interrupts mid-sentence” is a bad product behavior

In real sales calls, an AI cutting in mid-sentence is obnoxious. The founder will stumble. The judge may talk over it. Audio chaos.

Frame it as:

> “whispers at the next conversational beat”

or

> “nudges before you answer the objection.”

The demo can still feel real-time without literally stepping on every sentence.

## 3. Too many knowledge sources

Cut these for hackathon v1:

- competitor pitch archives
- prospect public statements
- “entire history”
- hostile VC question database
- live deck parsing
- public web search

Use preloaded artifacts. Make them look real. That is enough.

## 4. The “Loom in 2022” line is too magical

This sounds cool but risks feeling fake. Also now you need a historical pitch archive retrieval product.

Instead, make the second beat about the seller’s own prior calls:

> “You handled this with Acme last week.”

That is more believable, more useful, and easier to prove.

## 5. False positives are the death case

A copilot that interrupts unnecessarily is worse than useless.

Your product needs a visible “confidence gate”:

```txt
Interrupt threshold: 0.85
Current risk: 0.91
Action: Whisper
```

This makes the system feel disciplined.
